# 开发规则

## 项目结构
- 使用模块化结构（services / pages / models）
- 不要修改无关文件
- 每次只实现一个功能
- 所有网络请求必须封装在 services 层
- UI 和逻辑分离
- 不要引入复杂状态管理（先用简单方案）

## 代码要求
- 清晰、可运行
- 不要过度设计

## 强制规则

1. 只能修改 lib/ 目录
2. 禁止修改 android / ios / web / windows 等目录
3. 禁止修改 Gradle / Android 构建文件
4. pubspec.yaml 只能新增依赖，不能删除或升级已有依赖
5. 每次任务只修改一个功能模块
6. 禁止进行项目级重构或"优化整个项目"

## 安全原则

如果任务涉及 Flutter SDK / Android Gradle / 构建系统：
必须停止并提醒用户确认

## 输出要求

必须说明：
- 修改了哪些文件
- 做了什么改动
