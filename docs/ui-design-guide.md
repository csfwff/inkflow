# UI 设计指南

## 产品名称
Inkflow（移动端写作与发布工具）

## 产品定位
让用户可以在手机上：
- 写 Markdown 文章
- 上传图片到图床
- 一键发布到 GitHub 仓库（Hexo博客）

## 用户群体
- 开发者
- 博客写作者
- 技术记录用户

## 核心功能
1. Markdown 写作编辑器
2. 图片上传（支持多图床）
3. GitHub 发布（token配置）
4. 文章列表管理
5. 设置页（GitHub / 图床 / 编辑器）

## 页面设计

### 1. 首页（文章列表）
- 显示文章列表
- 新建按钮
- 草稿区

### 2. 编辑页
- Markdown 编辑区域
- 工具栏（插入图片 / 标题 / 发布）
- 实时预览切换

### 3. 发布页（可合并）
- GitHub 发布确认
- 选择分类 / tags

### 4. 设置页（重点）
- GitHub配置（token / repo）
- 图床配置（多选：Cloudinary / Imgur / OSS）
- 编辑器设置

### 5. 图床选择弹窗
- 多服务切换 UI

## UI 风格要求
- 极简风
- 类 Notion / Linear 风格
- 深色模式优先
- 有轻微毛玻璃 / 卡片设计
- 圆角 12~16px
- 强调"写作感"和"安静感"

## 交互要求
- 一键发布
- 上传进度反馈
- 表单验证
- 测试连接按钮（GitHub / 图床）

## 输出要求
1. 页面结构图（文字说明）
2. 每个页面 UI 布局描述
3. 组件设计（按钮 / 输入框 / 卡片）
4. 推荐配色方案
5. Flutter实现建议（可选）
